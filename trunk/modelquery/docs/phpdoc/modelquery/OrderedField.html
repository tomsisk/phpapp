Class OrderedField

Description
Description | Vars (details) | Methods (details)

Field contains an ordered integer.

OrderedField fields get special handling, because ModelQuery needs to manage the ordering to ensure there are no gaps or duplicates in the data store.

Example:

  1.  class Portfolio extends Model {
  2.      public function configure() {
  3.          $this->id = new IntegerField('ID', array('pk' => true));
  4.          $this->user = new ManyToOneField('User', 'User');
  5.          $this->sort_order = new OrderedField('Sort Order', array('user'));
  6.          $this->name = new CharField('ID', 100);
  7.      }
  8.  }
  9.  
  10.  // Start with a user
  11.  $user1 = $uq->get(1);
  12.  
  13.  // Create a few portofolios.  As we add them, ModelQuery assigns
  14.  // the sort_order field automatically.
  15.  $up1 = $pq->create(array('user' => $user1, 'title' = 'U1P1'));
  16.  $up1->save();
  17.  // $up1->sort_order is now "1"
  18.  $up2 = $pq->create(array('user' => $user1, 'title' = 'U1P2'));
  19.  $up2->save();
  20.  // $up2->sort_order is now "2"
  21.  $up3 = $pq->create(array('user' => $user1, 'title' = 'U1P3'));
  22.  $up3->save();
  23.  // $up3->sort_order is now "3"
  24.  
  25.  // What happens if we rearrange?
  26.  $up2->sort_order = 3;
  27.  $up2->save();
  28.  // $up2->sort_order is now "3"
  29.  // $up3->sort_order has been automatically updated to be "2"
  30.  
  31.  // ModelQuery fills in the gaps on delete
  32.  $up3->delete();
  33.  // $up2->sort_order is now "2" again
  34.  
  35.  // That groupBy field?  That tells ModelQuery how to determine
  36.  // unique ordering groups
  37.  // Let's look at a different user:
  38.  $user2 = $uq->get(2);
  39.  $up4 = $pq->create(array('user' => $user2, 'title' = 'U2P1'));
  40.  $up4->save();
  41.  // $up4->sort_order is now "1"
  42.  // $up1->sort_order is also still "1", because they belong to
  43.  // different users (and "user" is the "groupBy" parameter in the
  44.  // constructor)
