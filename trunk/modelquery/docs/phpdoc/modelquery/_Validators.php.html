Class Description
ModelValidator A validator that validates the entire model as a whole, rather than limited to individual fields.
ConditionalRequiredStringValidator Requires that if one field has a non-empty value, another field must also have a non-empty value.
FieldsMatchValidator Requires that two fields have matching values.
OneOrMoreValidator Require that one or more of the listed fields have non-empty values.
UniqueFieldsValidator Require that a set of fields be unique within all model instances.
FieldValidator A validator that checks individual field values.
RequiredFieldValidator Required that a field have a non-null value.
MinLengthValidator Required that a text field have a minimum length.
MaxLengthValidator Required that a text field have a maximum length.
UniqueFieldValidator Require that a field value be unique among all model instances.
RequiredStringValidator Require that a string have a non-empty value. This differs from RequiredFieldValidator in that while RequiredFieldValidator will accept an empty string as a non-null value, this validator will not.
NumericValidator Require that a field value be numeric.
PasswordValidator Imposes certain character requirements on password fields.
NonEmptyArrayValidator Require that an array field have a least one item in it.
DateValidator Required a valid date.
EmailValidator Require a valid email address format.
ConditionalValidator Conditionally includes a model validator depending on another field value. If the specified field matches the required value, then the additional validator will be added to the validation stack.
